# duckdb-example
This repository helps you get started with DuckDB for quick and easy SQL querying on small CSV datasets. It's perfect for when you need to run SQL queries on your data without setting up a big database server.

## Getting Started
Follow these simple steps to set up and use this project.

## Prerequisites
You'll need these installed on your computer:
- Python 3.9+: The programming language used.
- Poetry: A tool for managing Python projects and their dependencies.

## Setup
Clone the repository:
If you haven't already, download this project to your computer.

git clone https://github.com/victoremnm/duckdb-example
cd duckdb-example

Install dependencies:
Use Poetry to set up the project and install necessary libraries like duckdb and pandas.
```shell
poetry install --no-root
```

The --no-root flag tells Poetry to just install the required libraries, not to try and install this project as a reusable Python package.

## How to Use
This repository contains a sample CSV file (mock_data.csv) and a Python script (read_csv.py) that demonstrates how to load and query data using DuckDB.

Run the Python script:
The read_csv.py script will connect to a temporary DuckDB database, load your mock_data.csv into it, and then run some example SQL queries.

```shell
poetry run python read_csv.py
```
You can open read_csv.py to see the Python code that connects to DuckDB and executes SQL. You can modify this script to run your own queries.

Interactive SQL with DuckDB CLI (Optional):
If you want to play around with SQL directly, you can use the DuckDB Command Line Interface (CLI).

First, ensure you have the duckdb executable available. Poetry installs it in your virtual environment.

You can open a DuckDB in-memory database and load your CSV:
```
poetry run duckdb
.read read_csv.py  -- This will execute the Python script's SQL directly in the CLI (if it's pure SQL)
-- OR, more commonly, to load a CSV directly:
CREATE TABLE policies AS SELECT * FROM 'mock_data.csv';
SELECT * FROM policies LIMIT 5;
.quit
```

If you create a persistent DuckDB database file (e.g., my_data.duckdb in your Python script), you can open it directly with the CLI:
```
poetry run duckdb my_database.db
SELECT * FROM policies limit 10;
.quit
```

## Key Files
- mock_data.csv: Your sample dataset.
- read_csv.py: The main Python script that uses DuckDB.
- pyproject.toml: Poetry's configuration file for project dependencies.
- poetry.lock: Generated by Poetry, locks the exact versions of dependencies.
